Introduction

YALLA.NET is Yet Another Logging Library Abstraction for .NET.

Figure 1: Deployment diagram

 Note
This is the YALLA.NET Core documentation, which details the YALLA.NET Logging API. For YALLA.NET Application API please refer to the primary YALLA.NET documentation. For the node.js library please refer to https://github.com/kessler/node-yalla

Supported Platforms

  • .NET Framework 4
  • Silverlight 5
  • Windows 8 (WIP)
  • Windows Phone 8.1 (WIP)
  • Windows Phone Silverlight 8 (WIP)
  • Xamarin.Android (WIP)
  • Xamarin.iOS (WIP)
  • Xamarin.OSX (WIP)
  • Portable Class Libraries
 Note

This is a preliminary release with limited functionality. .NET Framework 4 (or later), Silverlight 5 and Portable Class Libraries are currently the only supported platforms.

Getting Started

As shown in Figure 1 above, YALLA.NET supports a variety of use cases, which include the following:

  1. Application Code Input: Your application code either requires the addition of logging functionality, or offers logging functionality that requires modification.

  2. Library Code Input: Your library code either requires the addition of logging functionality, or offers logging functionality that requires modification.

  3. Platform Facility Output: Your application or library code requires logging functionality with one or more platform-specific facility outputs.

 Note

This is a preliminary release with limited functionality.Application Code Input, Library Code Input and Platform Facility Output are the only use cases that are currently supported.

You only need to add the YALLA.NET Core as a dependency to your project in order to apply the Application Code Input use case or the Library Code Input use case. For all other use cases (e.g. producing YALLA.NET log(s) in your application) please refer to the primary YALLA.NET documentation.

In order to use YALLA.NET in your application or library, you need to add the YALLA.NET Core as a dependency to your project(s). If you haven't already, install the Yalla.Core NuGet package. The current version is Version 0.2.0.

ILogFactory is the focal point of YALLA.NET Logging API. If you are developing a library, you should expose an API that would allow its consumer to specify an ILogFactory instance during initialization. You may provide fallback functionality by getting the NoOpLogFactoryInstance.

A call to GetLogger(Type) will return the ILog for the specified type. A call to GetLogger(String) will return the ILog for the specified category. Please refer to the ILog type reference for YALLA.NET Logging API details.
